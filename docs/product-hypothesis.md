## 1. 출발점 — 보험 도메인의 첫인상 문제

보험 매칭/비교 서비스는 어떤 product를 만들어도 첫인상에 신뢰를 주기가 구조적으로 어려움. 일반인 입장에서는 신규 진입자를 기존 "보험 DB업"과 같은 frame으로 받아들임. 이 불신 frame은 4겹의 레이어로 쌓여 있음.

| 레이어 | 사용자 내적 발화 | 무너지는 순간 |
|---|---|---|
| 1. 정보 안전 | "내 번호 팔리는 거 아냐?" | 전화번호 입력 칸 보는 순간 |
| 2. 강매 공포 | "받으면 전화 폭탄 와" | "견적 받기" 버튼 누르는 순간 |
| 3. 중립성 의심 | "어차피 GA지" | "회사 소개" 가서 GA 라이선스 발견하는 순간 |
| 4. 결과 신뢰 | "추천된 거 진짜 최선이야?" | 제안서 받았는데 비교가 모호한 순간 |

**핵심 관찰**: 첫인상 전투는 레이어 1·2에서 결판. 더 많은 잠재 사용자는 3·4를 볼 기회조차 없이 1·2에서 이탈. 토스·카카오 같은 거대 신뢰 자본도 이걸 못 깨뜨림(카카오페이 비교추천 사용자 67만 vs 실제 계약 6.2만 = 9.3% conversion). **자본·브랜드로 정면 돌파하는 길은 막혀 있음.**

## 2. 제품 가설 (Tool-led entry)

### 2.1 가설 정의

> **첫 화면 = 도구**. 가입자가 보험 의사결정의 어느 지점에 있든 "혼자서도 답을 얻을 수 있는 도구"가 진입 자산. 매칭은 그 도구 안에서 사용자가 "이 결과를 가지고 진짜 견적을 받아보고 싶다"고 명시 트리거할 때만 활성화되는 보조 기능.

이게 의미하는 바:
- 랜딩 = 도구, 매칭은 메뉴 어딘가에 있는 옵션
- 도구가 traffic·SEO·brand를 모두 책임짐
- 매칭은 "도구로 분석을 끝낸 사용자"라는 lead quality가 압도적으로 높은 깔때기
- 사용자는 "이 회사는 보험을 이해하는 도구 회사"로 인식 → DB업 frame과 다른 frame이 만들어짐

### 2.2 시장의 4박자 (만족 체감 메커니즘)

한국 가입자가 "잘 가입했다"고 체감하는 4박자:
1. **강매 차단** — 구조적으로 콜백·강매가 일어나지 않음
2. **또래 비교** — 시장 평균 대비 내 위치가 보임
3. **구멍 명시** — 보장 갭과 함정이 정직하게 드러남
4. **가입 후 합리화** — 결정의 정당성이 시간이 지나도 확인됨

벤치마크 분석상 4박자를 동시에 충족하는 서비스는 글로벌 통틀어 없음. 특히 (1) 강매 차단의 구조적 증명이 가장 큰 빈자리.

### 2.3 차별화 포지션

| 축 | 기존 경쟁자 | 우리 |
|---|---|---|
| 진입 가치 | 매칭 (DB 수집 깔때기) | 도구 (PII 없이 즉시 가치) |
| 신뢰 약속 | "객관 비교" (지킬 수 없음) | "권유 검증" (지킬 수 있음) |
| Brand frame | 보험 매칭 플랫폼 | 보험 도구 회사 |
| 수익 인센티브 | 매칭 성공 시 100% | 도구 + B2B2C + 자문 + 매칭 (정렬됨) |
| 콜백 정책 | 견적 요청 = 콜백 동의 | 견적 요청 ≠ 콜백 동의 (구조적 분리) |

## 3. Target 사용자 세그먼트

| 세그먼트 | 비중 (추정) | 특징 | 우리에게 의미 |
|---|---|---|---|
| 트라우마 그룹 | ~20% | 이미 호구 경험. 어떤 보험 플랫폼도 안 믿음 | 포기. 시간이 약 |
| **경계 그룹** | **~60%** | 의심하지만 도구가 있으면 써봄. 휴대폰 번호 입력 직전 50% 이탈 | **진짜 타겟** |
| 적극 비교파 | ~20% | 이미 비교 앱 여러 개 써봄. 호갱 안 되려고 적극적 | 쉽게 유입되지만 이탈도 빠름 |

**경계 그룹**이 우리의 PMF 타겟. 이 세그먼트는 "값을 주면 한 발 더 내딛는" 그룹. 첫 화면에서 가입 권유 0 + 콜 차단 약속이 곧장 conversion으로 이어질 가능성.

세대별 우선 진입 채널 (참고):
- 20대: 토스·카카오페이·디시·블라인드
- 30대: 맘카페·재테크카페·유튜브
- 40대: 네이버 블로그·유튜브·설계사

상세 세그먼트 데이터는 [market-research.md §5](./market-research.md) 참조.

## 4. Funnel 설계

```
[랜딩]
"보험 갈아타기, 손익 계산해드려요. 5분이면 끝."
        ↓ (PII 0, 로그인 0)
[도구 사용 — 리모델링 손익 계산기]
사용자가 현재 보험 / 권유받은 신상품 정보 입력
        ↓
[결과 화면 — 한 줄 결론 + 8개 섹션]
- 결론 한 줄 + 신호등 점수
- 즉시 손실 / 30년 누적 / 보장 비교 / 함정 / 고지의무 / 권유 진실성
- 결과 저장·공유 (이메일/익명 링크, 휴대폰 번호 없이)
        ↓ (사용자가 명시 click)
[다음 단계 옵션 3개]
1. 현재 유지 → "1년 후 재점검 알림 받기"
2. 부분 변경 → "특약 정리만 도와줄 설계사 매칭하기"
3. 갈아타기 OK → "함정 없는 비슷한 상품 비교 견적 받아보기"
        ↓ (옵션 2 또는 3 선택 시 처음 매칭 등장)
[매칭 요청 작성]
- 분석 결과가 자동 spec으로 채워짐
- "전화 안 옴. 카톡 안 옴. 앱 안에서만" invariant 명시
- 휴대폰 번호 본인인증 1회만, 설계사 미공개
        ↓
[제안서 비교 화면]
- 정규화된 그리드 / 또래 평균 / 함정 자동 라벨
- 거절 1탭 + 거절 사유 누적
        ↓
[가입 후 합리화 루프]
- 가입 결정 요약 카드 (공유 가능 artifact)
- 3개월 / 1년 후 재진단 푸시
- 청구 시점 자동 알림
```

### 4.1 핵심 invariant
- **휴대폰 번호는 funnel의 끝**. 도구 사용 시점에는 절대 수집 안 함
- **매칭은 선택지 3개 중 1개**. 도구가 BM이지 매칭이 BM이 아니라는 메시지
- **"제안 받기 ≠ 콜백 동의"**. 전화는 가입자가 "통화 요청" 버튼을 눌러야만 발생. 서버에서 콜·SMS 시도 차단, 시도 시 페널티 카운트

### 4.2 Layer별 사용자 control

| Tier | 데이터 노출 | 사용자 동의 |
|---|---|---|
| 0. 익명 도구 | PII 0 | 없음 |
| 1. 본인 데이터 연동 | 본인인증 (내보험찾아줌) | 명시 |
| 2. 익명 견적 받기 | 설계사는 가입자 PII 안 봄 | 명시 |
| 3. 1:1 매칭 | 휴대폰 번호 본인인증 | 명시 |

## 5. Hero Product — 리모델링 손익 계산기

### 5.1 선택 근거

가입자가 "지금 당장 답이 필요하다"고 느끼는 8가지 순간을 mining한 결과, 3개 archetype으로 묶임:
- A: "이 권유 검증해줘" (청약서 검토 / 갈아타기 / 갱신 인상 / 단기납·무해지)
- B: "내 위치 알려줘" (또래 비교 / 부모 양도)
- C: "내 케이스에 답 줘" (임산부 데드라인 / 청구 가능성)

Archetype A가 가장 강한 이유:
- 시장 trauma 매우 큼 (부당승환 민원 분기 +54% 폭증)
- 사용자 intent 단일·명확
- 검색 행동 강함 ("리모델링 후회", "단기납 후기" 폭검색)
- 매칭으로 양방향 전환

Archetype A 안에서 hero 선택: **리모델링 손익 계산기 (Sub-A1)**
- 시장 trauma 가장 폭발적 (+54%)
- 결과가 양방향 모두 가치 ("갈아타지 마세요"도 trust 모먼트)
- 매칭 단가 최대 (액티브 가입 의도자)
- 모든 세대에 hit (부모/배우자 권유 케이스가 모든 세대에 있음)

Hero 카피 후보:
- "보험 갈아타기, 손익 계산해드려요. 5분이면 끝."
- "설계사가 갈아타라고 했나요? 사인 전에 검증해드려요."
- "리모델링 권유, 진짜 내게 이득인지 무료로 확인하세요."

### 5.2 사용자의 5가지 진짜 질문

가입자가 리모델링 권유받고 들어올 때 머릿속에 있는 질문:
1. "지금 깨면 얼마 손해 봐?" (해지환급 손실)
2. "새거가 진짜 더 나아?" (보장 + 30년 누적)
3. "중간에 보장 빵꾸 안 나?" (면책·감액기간 공백)
4. "내가 새거 가입할 자격은 돼?" (고지의무 위험)
5. "설계사가 진짜 내 편이야 자기 수수료야?" (권유 진실성)

이 5개 질문에 한 화면에서 답을 주는 게 도구의 mission.

### 5.3 입력 — Progressive Disclosure

#### Step 1 — 빠른 입력 (1분, 결과 대략)
- 현재 보험: 보험사 / 상품 카테고리(암/종신/실손/CI 등) / 가입 시점 / 월 보험료 / 현재 나이
- 권유 신상품: 월 보험료 / 카테고리 / 보장기간 / 갱신/비갱신
- 출력: 대략적 한 줄 결론 + "정확한 답 보려면 정보 더 입력하시겠어요?"

#### Step 2 — 정확한 입력 (5분, 결과 정밀)
- 현재 증권 사진/PDF 업로드 (OCR + LLM 자동 파싱) — 또는 내보험찾아줌 연동(옵션)
- 신상품 설계 제안서 PDF 업로드 (LLM 자동 파싱)
- 자동 채워진 필드를 사용자가 확인·수정

원칙: 휴대폰 번호·이름 입력 없음. Step 1만으로도 가치 있는 답 출력.

### 5.4 결과 화면 — 8개 섹션

위에서 아래로 스크롤하며 디테일이 깊어지는 구조. 상단 fold에서 결론이 끝나야 함.

#### Section 1 — 한 줄 결론 (상단 fold)
- 신호등 + 점수 (예: 빨강 18/100 "갈아타지 마세요")
- 한 줄 카피 (예: "갈아타면 30년간 누적 1,847만 원 손해예요")
- 3줄 핵심 근거 (가장 큰 손실 항목 3개 자동 추출)
- 3개 액션 버튼: [현재 유지하기] [부분 변경만 보기] [그래도 갈아타기 옵션 보기]

이 한 화면에서 약 80% 사용자는 결정 끝남.

#### Section 2 — 즉시 손실 (Now)
| 항목 | 표시 |
|---|---|
| 누적 납입 보험료 | 예: 12,400,000원 (104개월 × 119,200원) |
| 현재 해지환급금 | 예: 7,580,000원 |
| **즉시 손실** | **−4,820,000원** (빨강, 큰 글씨) |
| 손실률 | 납입 보험료 대비 38.9% |

추가:
- "약관대출 가능액 X만 원도 사라집니다" 표시
- 무해지환급형 경고: "이건 무해지환급형이라 즉시 손실 100%예요"
- 세제 영향: "기타소득세 16.5% 추가 부담 가능"

#### Section 3 — 30년 누적 비교 (Long term)
**그래프 1개가 핵심.** 30년 누적 비용 곡선:
- X축: 나이 (현재 ~ 85세)
- Y축: 누적 납입 보험료
- 라인 2개: "현재 유지" vs "갈아타기"
- 갱신형이면 갱신 시점마다 계단 상승 시각화
- 손익분기점 명시 (예: "72세에 갈아타기가 누적 이득으로 전환됨")

옆 숫자 박스 3개:
- 65세 시점 누적 차이
- 80세 시점 누적 차이
- 85세 시점 최종 누적 차이 (가장 큰 글씨)

추가:
- 갱신형 인상 시뮬 가정 명시: "5년마다 평균 +18% 인상으로 가정. 실제 인상폭은 다를 수 있습니다"
- 부분 변경 시나리오: "특약 X만 정리하면 누적 -340만 원" — 3rd option

#### Section 4 — 보장 정밀 비교

##### 4-1. 보장 항목 그리드 (정규화)
| 보장 항목 | 현재 | 신상품 | 차이 |
|---|---|---|---|
| 일반암 진단비 | 5,000만 | 7,000만 | ↑ +2,000만 |
| 유사암 비율 | 20% | 10% | ↓ 소액암 받을 돈 줄어듦 |
| 뇌혈관 진단비 코드 범위 | I60–I69 | I60–I62 | 뇌경색 빠짐 |
| 갱신주기 | 비갱신 | 5년 갱신 | 미래 인상 위험 |
| 면책기간 | 끝남 | 90일 신규 | 공백 |
| 감액기간 | 끝남 | 1년 50% | 공백 |
| 환급률 (만기) | 50% | 0% (무해지) | 환급 없음 |

색깔 코딩: ↑ 녹색, ↓ 빨강·노랑.

##### 4-2. 함정 라벨 자동 추출 (신상품)
- "무해지환급형" — 중도 해지 시 환급금 0원. 7년 유지율 60%
- "5년 갱신형" — 60세 시점 보험료 약 X원 예상
- "감액기간 1년" — 가입 후 1년간 진단 시 50%만 지급
- "유사암 10%" — 갑상선·제자리암 진단 시 약속 금액의 1/10

##### 4-3. **옛 상품의 절판 강점 (핵심 차별화)**
신상품 권유 시 가입자가 가장 놓치는 부분. 옛 상품에만 있고 신상품엔 없는 보장. 예시:
- "1세대 실손 — 자기부담 0%, 다시 가입 불가능"
- "가입 당시 보장 코드 I60–I69 — 현재 신상품은 대부분 I60–I62로 축소"
- "비갱신형 100세 만기 — 같은 조건 현재 시장에서 월 X원 비쌈"

이게 도구의 가장 강한 trust 모먼트. "당신 옛 보험에 좋은 게 있어요"라고 말해주는 첫 도구.

#### Section 5 — 새 가입 자격 진단 (고지의무 위험)
입력: 최근 5년 내 진료·복용·검사 받은 적 있는지 체크박스 (PII 0)
출력:
- "고지의무 사항 X개 발견됨"
- 각 항목이 ① 부담보 ② 보험료 할증 ③ 가입 거절 중 어디일 가능성
- 한 줄: "신상품에서 OO이 부담보 처리될 가능성 높음 — 정작 가장 필요한 보장이 빠질 위험"
- Disclaimer: "참고용. 실제 인수 심사는 보험사 결정"

가입자 70%가 모르는 risk → "이거 입력 안 했으면 큰일 날 뻔" 모먼트 = trust 폭발.

#### Section 6 — 권유 진실성 진단
톤: 공격 X, 재확인 톤.

##### 6-1. 권유 신호 분석
입력: 설계사가 한 말 중 해당되는 것 체크
- "곧 단종/절판입니다"
- "이번 달까지만 가능"
- "사은품 드려요"
- "환급률 100% 이상"
- "예금/적금보다 낫다"

출력 (신호 개수 따라):
- 0개: "건전한 권유 신호"
- 1–2개: "재확인 권장"
- 3개 이상: "절판/단기납 마케팅 패턴 — 금감원 소비자경보 발령 사례"

##### 6-2. 시장 맥락 정보
- "현재 단기납 종신 절판 마케팅 활발. 금감원 2024 소비자경보 발령" (link)
- "부당승환 민원이 직전 분기 +54%"
- 신규 가입 시 설계사 수수료 신규 발생 사실 명시 (참고 정보, 비난조 X)

#### Section 7 — 추천 액션 3개 옵션
| 옵션 | 30년 누적 영향 | 권장도 |
|---|---|---|
| A. 현재 유지 | 기준선 | 녹색 — 가장 안전 |
| B. 부분 변경 (특약 X만 정리) | +120만 원 절약 | 노랑 — 합리적 |
| C. 완전 갈아타기 | −1,847만 원 손해 | 빨강 — 비권장 |

각 옵션마다 [이 옵션으로 가기] 버튼.

#### Section 8 — Next step (매칭 트리거, optional)
화면 가장 아래. 강요 X.
- 녹색: "현재 유지 결정하셨네요. 1년 후 재점검 알림 받기" (이메일/익명 토큰)
- 노랑: "부분 변경하시겠어요? 특약 정리만 도와줄 설계사 매칭하기"
- 빨강: "그래도 갈아타시겠다면, 함정 없는 비슷한 상품 비교 견적 받아보기"

여기서 처음 매칭 등장. 진입 시 "전화 안 옴. 카톡 안 옴. 앱 안에서만" invariant 카피 노출.

### 5.5 신뢰 디자인 6원칙

1. **계산 가정 항상 명시** — "5년마다 +18% 인상 가정"
2. **데이터 출처 표기** — "손보협 공시 데이터 기준"
3. **확률·범위로 표시** — 단정 대신 "1,500–2,100만 원 손해 범위 추정"
4. **모르는 부분 솔직히** — "이 항목은 약관 추가 확인 필요"
5. **계산 다시 보기** — 클릭 시 계산식과 가정 펼침
6. **분석 결과 PDF 저장/공유** — 가족·다른 설계사에게 보여줄 artifact

### 5.6 Edge cases

| 상황 | 대응 |
|---|---|
| 사용자가 옛 보험 정보 모름 | 보험사 + 상품명만으로 공시 약관 매칭. 부족하면 내보험찾아줌 연동 옵션 |
| 신상품 정보가 설계 제안서 PDF만 있음 | PDF 업로드 → LLM 파싱 → 사용자 확인·수정 |
| 1·2세대 실손 같은 절판 상품 | 자동 감지 + "재가입 불가" 강력 경고 |
| 사용자가 이미 무해지환급형 가입자 | 즉시 손실 100% 명시 + 그동안 낸 돈 전액 손실 시각화 |

### 5.7 MVP 범위 (제안)

8개 섹션 중 v1 우선:
- **MVP**: Section 1 (결론), Section 3 (30년 누적 그래프), Section 4 (보장 비교 + 함정 + 절판 강점)
- v1.1: Section 2 (즉시 손실), Section 7 (추천 액션 3개)
- v2: Section 5 (고지의무), Section 6 (권유 진실성)
- v2+: Section 8 (매칭 트리거)

MVP 카테고리 우선순위는 [Open Questions](#10-open-questions) 참조.

## 6. Supporting Products (v2 이후)

Archetype A 안의 다른 도구들. Hero (Sub-A1)이 안정화된 후 확장.

### Sub-A2: 청약서 사인 직전 함정 검사
- 입력: 청약서 PDF 또는 설계 제안서 이미지/PDF
- 출력: 함정 자동 라벨 + 보험료 적정성(또래 평균 대비) + 동일 보장의 다른 보험사 가격 + "사인 OK" / "X개 항목 재확인 필요"
- 시장 trauma: 단기납 종신 / 무해지 분쟁
- 진입 마찰: 매우 낮음 (PDF 1장)

### Sub-A3: 갱신 인상 통보 대응
- 입력: 보험사 갱신 인상 안내문 + 기존 보험 정보
- 출력: 인상폭의 시장 평균 대비 정상 여부 / 해지 손실 계산 / 유지·해지·부분 변경 3옵션의 30년 누적 비교
- 시장 trauma: 4세대 실손 +20%, 비급여 +300% 시즌

### v3+ 도구 suite 확장
- 시나리오 진단 ("암 걸리면 얼마", "죽으면 가족이 얼마")
- 또래 비교 대시보드
- 청구 가능성 진단

### Brand promise 일관성

모든 도구가 **"권유 검증"** 또는 **"내 보험 검증"** 이라는 단일 brand promise 아래 통일. 사용자는 "이 회사는 내가 받은 권유를 의심 없이 검증해주는 곳"으로 인식.

## 7. Defensibility / Moat

| 위협 | 우리 방어선 |
|---|---|
| 보맵·굿리치가 LLM 약관 분석 출시 | 우리는 처음부터 도구 회사. 그들은 "GA + 도구"로 priority 충돌 |
| 토스·카카오가 같은 도구 출시 | 가능. 그러나 그들의 트래픽은 자보, 약관 분석은 보장성 — 트래픽 소스 misalign |
| 보험사가 직접 약관 풀이 사이트 | 자기 상품만 풀이 → 비교 불가. 우리는 cross-company라 가치 우위 |
| 신규 스타트업이 같은 모델로 진입 | moat = (1) 누적 약관 DB + 분석 정확도, (2) SEO 자산 누적, (3) "권유 검증 brand" 선점 |

### Moat 후보 2가지
1. **약관 분석 정확도 vertical** — 약관 + 의료 코드(KCD) + 보험사 분쟁사례를 fine-tune한 도메인 모델. 단순 GPT 호출과의 격차
2. **누적 사용자 데이터** — 누가 어떤 보험을 가지고 있고, 어떤 갭이 있고, 어떤 매칭 결과에 만족했는지의 longitudinal 데이터. 또래 비교 정확도가 사용자 수에 비례

## 8. 비즈니스 모델

기존 매칭 수수료 100% 모델은 funnel을 짧게 만들고 강매를 권장하는 인센티브. 가설 B는 이걸 의도적으로 깸 → 수익 다각화 필수.

### 단기 (1년)
- 매칭 수수료 (성공 보수) — 도구 기반 lead quality 상승으로 단가 ↑
- B2B2C 회사 복지 채널 — 회사가 도구 라이센스 구매, 직원은 무료
- 유료 자문 결합 — 손해사정사·변호사·CFP 1:1 컨설팅 (도구 → 자문 → 매칭의 3단)

### 중기 (1–3년)
- Premium 도구 (월 구독) — 가족 전체 통합관리, 청구 어시스턴트, 갱신 알람
- 설계사용 B2B SaaS — 약관 분석 엔진을 설계사 영업 도구로 (GA 인센티브와 분리)
- 데이터 인사이트 (anonymized) — 보험사가 자기 약관 경쟁력 진단

### 장기 moat
도구 + 데이터 + brand 결합으로 매칭 수수료 비중을 50% 이하로 낮추면 **사용자가 직접 지불하는 SaaS** 전환 가능. 보험 도메인의 "Notion" 포지션.

### 인센티브 정렬
매출 중 매칭 비중이 50% 미만이 되면 우리 인센티브가 사용자와 정렬됨. "이 회사는 진짜 가입자 편"이 product mechanics가 아니라 매출 구조로 증명됨.

## 9. Risk와 Mitigation

| Risk | 가능성 | 영향 | Mitigation |
|---|---|---|---|
| LLM이 약관 잘못 분석 → 가입자 손해 | 중 | 매우 큼 | (1) "참고용, 최종 판단은 설계사·손해사정사" disclaimer, (2) high-confidence 영역만 단정적 표시, 모호한 건 "전문가 자문 권장" 라벨, (3) 보험사 공식 약관 DB 구축으로 환각 제거 |
| 도구 만들고 트래픽 안 옴 | 중 | 큼 | SEO 자산화 (상품별 페이지) + 콘텐츠 마케팅 + 커뮤니티 seeding (디시 보험갤·맘카페에 결과 공유 artifact) |
| 매칭 conversion 낮아 단기 매출 부재 | 높음 | 큼 | B2B2C 회사 복지 매출 cushion. 유료 자문을 도구↔매칭 사이 단으로 |
| 보맵·굿리치가 6개월 안에 따라옴 | 중 | 중 | "도구 회사" brand 선점. 그들은 매칭 brand로 pivot 어려움 |
| 보험사 약관 PDF 저작권/접근 이슈 | 낮음 | 중 | 가입자 본인 증권 업로드는 합법. 우리 DB는 협회 공시 + 사용자 기여 기반 |
| 도구 정확도 검증 주체 부재 | 중 | 중 | CFP/손해사정사 advisory board, 외부 감사 분기별, 잘못된 분석 신고 보상 정책 |
| 부당승환 권유 도구로 오인 | 낮음 | 큼 | 결과의 50%+ 케이스가 "유지 권장"이 나와야 정상. 매칭 conversion에 KPI 종속 X |

## 10. Open Questions

다음 단계로 진행하려면 결정 필요한 사항.

### Q1. MVP 카테고리 우선순위
종신·암·실손·CI 중 어느 상품군부터 정밀하게 풀지.
- 후보: **단기납 종신**(절판 마케팅 trauma) + **실손 4세대 전환**(전환 압박)
- 결정 보류

### Q2. 계산 가정 출처
갱신 인상률, 평균 해지환급률 등의 가정 데이터 출처:
- 옵션 A: 손보협 공시 hard data만 사용 (정확성 ↑, 출시 시간 ↑)
- 옵션 B: 시장 평균 추정 + 가정 명시 (빠른 출시, 신뢰는 transparency로)
- 결정 보류

### Q3. 결과 공유 artifact의 default 형식
- 후보: PDF / 카드 이미지(맘카페·디시 공유용) / 익명 링크
- 추천: 익명 링크 + 옵션으로 PDF·이미지 다운로드
- 결정 보류

### Q4. 현재 product의 변경 범위
가설 B는 "현재 매칭 중심 funnel → 도구 중심 funnel"로 information architecture 전체를 뒤집는 것. 점진적 추가가 아니라 reorganization.
- 갈아엎을 수 있는 범위 미정 → 실행 plan 영향
- 결정 보류

### Q5. Hero 도구 다음 단계
Sub-A1 안정화 후 Sub-A2(청약서 검사) / Sub-A3(갱신 대응) 중 어디 먼저?
- 후순위 결정

### Q6. 단기 매출 cushion
B는 6–12개월 매칭 conversion이 낮을 가능성 큼. runway 또는 다른 매출원(B2B2C, 유료 자문) 확보 가능 여부.
- 결정 보류
